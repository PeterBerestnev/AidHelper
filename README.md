# AidHelper - Telegram бот для отслеживания приема препарата

Telegram бот для отслеживания приема лекарственных препаратов с напоминаниями и историей приемов.

## Возможности

- 💊 Добавление записей о приеме препарата (количество таблеток и дозировка)
- ⚙️ Настройка курса приема (количество таблеток в день, дозировка, даты начала и окончания)
- 📊 Отслеживание прогресса курса с сравнением плана и фактического приема
- 📅 Просмотр приемов за сегодня с сравнением с планом
- 📈 История приемов за последние 7 дней
- ⏰ Напоминания о необходимости приема препарата с подтверждением
- 📄 Автоматическая отправка статистики в виде CSV файла при напоминании
- ✅ Подтверждение приема препарата прямо из напоминания (без подтверждения прием не засчитывается)
- 💾 Автоматическое сохранение данных в базу данных SQLite

## Установка

### Вариант 1: Запуск в Docker (рекомендуется)

**Требования:**
- Docker
- Docker Compose
- Make (опционально, для удобства)

1. Клонируйте репозиторий или скачайте файлы проекта

2. Создайте файл `.env`:
```bash
cp .env.example .env
```

3. Настройте `.env` файл:

   **Получение TELEGRAM_BOT_TOKEN:**
   1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
   2. Отправьте команду `/newbot`
   3. Следуйте инструкциям: укажите имя бота (например, "My Medication Bot")
   4. Укажите username бота (должен заканчиваться на `bot`, например: `my_medication_bot`)
   5. После создания бота BotFather отправит вам токен в формате `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`
   6. Скопируйте этот токен
   
   **Как найти своего бота в Telegram:**
   - В поиске Telegram введите `@ваш_username_бота` (например: `@my_medication_bot`)
   - Или откройте ссылку: `https://t.me/ваш_username_бота` (например: `https://t.me/my_medication_bot`)
   - После запуска бота (см. шаг 4) отправьте ему команду `/start` для начала работы

   **Получение USER_ID:**
   1. Откройте Telegram и найдите бота [@userinfobot](https://t.me/userinfobot)
   2. Отправьте команду `/start`
   3. Бот отправит вам информацию о вашем аккаунте, включая ваш User ID (число, например: `123456789`)
   4. Скопируйте это число

   **Заполнение `.env` файла:**
   Откройте файл `.env` и замените значения:
   ```
   TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
   USER_ID=ваш_user_id_от_userinfobot
   ```
   
   Пример:
   ```
   TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
   USER_ID=123456789
   ```

   **Прокси (опционально):**
   - Если соединение с `api.telegram.org` идет через прокси, укажите параметры в `.env`.
   - Поддерживаются переменные:
     - `HTTP_PROXY_URL` (например `http://user:pass@host:port`)
     - `HTTPS_PROXY_URL` (если не задано, берется `HTTP_PROXY_URL`)
     - `ALL_PROXY_URL` (например `socks5://user:pass@host:port` — если прокси задаётся универсально)
     - или общий `PROXY_URL` (то же самое для обоих протоколов)
     - `NO_PROXY` (например `api.telegram.org,localhost`)

4. Запустите бота с помощью Makefile:
```bash
make build    # Собрать Docker образ
make up       # Запустить контейнер
```

Или используйте docker-compose напрямую:
```bash
docker-compose build
docker-compose up -d
```

**Полезные команды Makefile:**
- `make help` - Показать все доступные команды
- `make build` - Собрать Docker образ
- `make up` - Запустить контейнер в фоновом режиме
- `make down` - Остановить и удалить контейнер
- `make logs` - Показать логи контейнера
- `make restart` - Перезапустить контейнер
- `make status` - Показать статус контейнера
- `make rebuild` - Пересобрать образ и перезапустить

### Вариант 2: Локальный запуск

1. Клонируйте репозиторий или скачайте файлы проекта

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

4. Настройте `.env` файл:
   - Получите токен бота у [@BotFather](https://t.me/BotFather) в Telegram (см. подробную инструкцию выше)
   - Получите свой User ID у [@userinfobot](https://t.me/userinfobot) в Telegram (см. подробную инструкцию выше)
   - Заполните значения в `.env` файле

5. Запустите бота:
```bash
python bot.py
```

## Использование

После запуска бота, отправьте команду `/start` в Telegram. Бот покажет главное меню с кнопками для удобной навигации.

**Все команды доступны двумя способами:**
- Через кнопки меню (рекомендуется)
- Через текстовые команды (например, `/add`, `/today`)

### Доступные команды:

**Главное меню с кнопками:**
- После команды `/start` появится меню с кнопками для всех основных функций
- Кнопка "📋 Меню" или команда `/menu` - показать главное меню

**Основные функции (доступны через кнопки или команды):**
- `💊 Добавить прием` или `/add` - Добавить запись о приеме препарата
- `📅 Сегодня` или `/today` - Показать все приемы за сегодня (с сравнением с планом)
- `📊 История` или `/history` - Показать сводку за последние 7 дней
- `⚙️ Настройки` или `/settings` - Настроить курс приема (количество таблеток в день, дозировка, даты)
- `📈 Прогресс` или `/progress` - Показать прогресс выполнения курса
- `⏰ Напоминания` или `/reminder` - Управление напоминаниями
- `❓ Помощь` или `/help` - Показать справку по командам

### Пример использования:

1. Настроить курс приема (рекомендуется сделать в первую очередь):
   - Отправьте `/settings`
   - Введите количество таблеток в день (например: 2)
   - Введите дозировку одной таблетки в мг (например: 500)
   - Введите дату начала курса в формате ГГГГ-ММ-ДД или "сегодня"
   - Введите дату окончания курса или "нет" для неограниченного курса

2. Добавить прием препарата:
   - Отправьте `/add`
   - Введите количество таблеток (например: 2)
   - Введите дозировку одной таблетки в мг (например: 500)

3. Посмотреть приемы за сегодня:
   - Отправьте `/today`
   - Бот покажет все приемы за сегодня и сравнение с планом

4. Посмотреть прогресс курса:
   - Отправьте `/progress`
   - Бот покажет статистику выполнения плана за весь период курса

5. Добавить напоминание:
   - Отправьте `/reminder`
   - Нажмите "➕ Добавить напоминание"
   - Введите время в формате HH:MM (например: 09:30)

6. Подтверждение приема из напоминания:
   - Когда придет напоминание, вместе с ним придет файл со статистикой за последние 30 дней
   - В напоминании будет показан план на сегодня и сколько уже принято
   - Нажмите кнопку "✅ Подтвердить прием" в напоминании
   - Введите количество таблеток и дозировку
   - Прием будет засчитан только после подтверждения

## Примечания

- База данных `medication.db` создается автоматически при первом запуске
- При использовании Docker база данных сохраняется в директории `data/` на хосте
- Напоминания проверяются каждую минуту
- При каждом напоминании автоматически отправляется CSV файл со статистикой за последние 30 дней

- Все данные хранятся локально в файле базы данных
- Контейнер автоматически перезапускается при сбое (restart: unless-stopped)

## Работа с несколькими препаратами

- Бот поддерживает несколько препаратов. По умолчанию все существующие данные относятся к препарату "Акнекутан".
- Текущий препарат выбирается **отдельно для каждого пользователя**.

### Как выбрать или добавить препарат

1. Нажмите кнопку `🧴 Препараты` в главном меню (или отправьте команду `/medications`).
2. Вы увидите список препаратов:
   - Текущий препарат помечен значком `✅`.
   - Остальные препараты помечены `⚪️`.
3. Для выбора другого препарата нажмите кнопку `Выбрать <название>`.
4. Для добавления нового препарата:
   - Нажмите кнопку `➕ Добавить препарат`.
   - Введите название препарата в ответ на сообщение.
   - Новый препарат будет создан и автоматически выбран текущим.

### На что влияет выбор текущего препарата

Все основные функции работают **в контексте текущего выбранного препарата**:

- `💊 Добавить прием` / `/add` — запись о приеме создается для текущего препарата.
- `📅 Сегодня` / `/today` — показываются только приемы текущего препарата.
- `📊 История` / `/history` — история и общая статистика выводятся только по текущему препарату.
- `⚙️ Настройки` / `/settings` — курс (таблетки в день, дозировка, даты) задается для текущего препарата.
- `📈 Прогресс` / `/progress` — прогресс считается относительно курса текущего препарата.
- `⏰ Напоминания` / `/reminder` — напоминания создаются и показываются для текущего препарата; текст напоминаний содержит его название.

### Обратная совместимость

- При первом запуске с новой версией бота автоматически создается препарат "Акнекутан".
- Все существующие записи о приеме, напоминания и настройки курса привязываются к "Акнекутану".
- Если вам нужен только один препарат, можно продолжать работать как раньше, просто считая, что это курс по "Акнекутану".
