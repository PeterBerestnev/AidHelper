# AidHelper — миграции и обновления

Этот файл описывает, какие шаги нужно выполнить при обновлении схемы базы данных и логики бота.

## Переход на версию с поддержкой нескольких препаратов

В этой версии добавлены:
- Таблица `medications` для хранения списка препаратов
- Поле `medication_id` в таблицах `medication_log`, `reminders`, `course_settings`
- Автоматический сид препарата по умолчанию `Акнекутан`

### Что делает миграция автоматически

При первом запуске **новой версии бота**:

1. Создается таблица `medications`, если её еще нет.
2. Добавляется препарат по умолчанию:
   - `Акнекутан` (если он еще не существует в таблице `medications`).
3. В таблицы `medication_log`, `reminders`, `course_settings` добавляется колонка `medication_id` (если её еще не было).
4. Всем существующим записям без `medication_id` автоматически проставляется `medication_id` препарата `Акнекутан`.

### Что нужно сделать пользователю

#### Вариант 1: Запуск через Docker (рекомендуется)

Если вы уже используете Docker и `docker-compose`:

1. Убедитесь, что контейнер остановлен (опционально):
   ```bash
   make down
   ```

2. Пересоберите образ и перезапустите контейнер:
   ```bash
   make rebuild
   ```

3. При старте контейнера миграция будет выполнена автоматически. 
   База данных `data/medication.db` будет обновлена на месте.

> **Важно:** перед обновлением можно сделать резервную копию базы:
> ```bash
> cp data/medication.db data/medication_backup.db
> ```

#### Вариант 2: Локальный запуск (без Docker)

1. Остановите старый процесс бота (если он запущен).
2. Установите/обновите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Запустите бота:
   ```bash
   python bot.py
   ```

При первом запуске новой версии бота:
- Схема базы данных будет автоматически обновлена.
- Все старые записи будут привязаны к препарату `Акнекутан`.

### Проверка успешной миграции

После обновления:

1. Запустите бота и отправьте команду `/medications` или нажмите кнопку `🧴 Препараты`.
2. В списке препаратов вы должны увидеть как минимум один препарат:
   - `Акнекутан` с отметкой `✅` (текущий препарат по умолчанию).
3. Команда `/history` (или кнопка `📊 История`) должна показать историю именно для `Акнекутана`.

Если что-то пошло не так:
- Проверьте логи контейнера/приложения
- При необходимости восстановите резервную копию `data/medication_backup.db` и повторите шаги.
